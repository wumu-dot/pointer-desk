# TFT 从 STM32 3.3V 取电掉压至 2.5V

- **Bug ID**：BUG-20260623-003
- **严重等级**：P1-严重
- **发现日期**：2026-06-23
- **修复日期**：2026-06-23

## 现象描述
TFT 屏幕亮度不稳定，偶尔闪烁。万用表量 TFT VCC = 2.5V，低于 ST7735S 最低工作电压 2.7V。

## 复现步骤
1. TFT VCC 接 STM32 板上 3.3V 排针
2. STM32 通过 USB 供电（限流 500mA）
3. 电机运转时 TFT 更暗

## 根因分析
USB 供电限流 500mA → STM32 LDO 输出有限 → TFT（背光吃电约 50mA）+ A4988 逻辑 + 面包板一起从 STM32 3.3V 取电，LDO 输出不足，电压掉至 2.5V。

## 修复方案
TFT VCC 改从适配器 3.3V 独立供电（适配器 3.3V 通道有余量）。A4988 VDD（微安级）可保持从适配器 3.3V 取电不变。

## 影响文件
- 硬件接线，无代码修改

## 验证方式
TFT VCC 改接适配器 3.3V → 万用表量 TFT VCC = 3.3V 稳定 → 电机运转时 TFT 不闪
